流媒体音乐平台 Spotify 及其优雅的食用方法

前言

自从网易云音乐把各种莫名奇妙的歌塞到我的音乐云盘之后，我就注销掉网易账户了。开什么玩笑，当版权侵权时候，就由用户来为网易背锅？另外国内公司当时也在各种版权大战，对于我这样靠随机播放来发现对味歌的人，完全不能接受这个歌不能听，那个歌需要单独付费的情况。于是我转向了国外平台。

毫无疑问，全球最大的流媒体音乐平台 Spotify 就进入了我的眼睛, 而经过这几年的使用，完全能满足我的需求。（其实另外还有个无损音乐平台叫 Tidal￼,可能国内98%的人都不知道，而我试用后还是觉得 Spotify 更方便，也许也是因为我不是个对音质十分挑剔的人。）

iOS

有意思的是 iOS 端的 Spotify 客户端中并不支持购买订阅，很有可能是为了避免苹果抽水。

另外，不习惯英文界面的朋友，在 iOS 设置里添加第二语言为繁体中文，打开 Spotify 的界面就变成了繁体了。（虽然对我都没差啦。）

如果作为一个免费用户，听歌是可以的，只是多了很多限制，比如：

  • 每小时切歌次数只有 6 次
  • 每听半小时就会有音频广告，需要点击广告才能于接下来 2 小时中无广告体验
  • 只能随机播放专辑中的歌曲
  • 歌曲串流不是极致音质
  • 等等

总的来说就是逼你开会员。

作为一个真•白嫖党，怎么可能就这样放弃呢。自从开始折腾 iOS 平台的软件，就发现有很多有意思的软件，可以利用他们来实现去广告，以增强体验。他们的共性之一就是利用自行生成证书，来实现中间人攻击(MitM)。共性之二就是 AppStore 中国大陆区并没有这些应用，需切换到外区。

  • 无音频广告
  • 无视频广告
  • 无限切歌
  • 可顺序播放
  • 不能下载音乐（这个无法破解）

Android

第三方魔改版

作为全球最大流媒体平台，自然是有被各种针对的。所以在各个破解软件网址里都能找到其破解版。我通常是在 mobilism 论坛或者 Telegram 频道 Modded Central Channel 上找的。作者有多个，不同作者魔改版并不能覆盖安装，所以自己找准一个作者的就行。当然，开放自然就有风险，包含且不限于银行账户、通信软件记录等敏感信息泄露，手机被抓为肉鸡用于挖矿。

自行下载，风险自负

效果

  • 无音频广告
  • 无视频广告
  • 无限切歌
  • 可顺序播放
  • 解锁 very high 音质（听不出来到底是不是真解锁了）
  • 不能下载音乐（这个无法破解）

PC端

以前我有用客户端来播放，然而找了很多方案，包括 hosts 屏蔽，静音广告时间及 AdGuard Home 等方法，均不满意。结合现在 iOS 端需要 MitM 才能屏蔽广告，我猜大概也需要在路由中安装证书，通过拦截 https 流量才能去除广告了。但这个方法可能带来不方便的是：

  1. 占用路由器性能，特别是硬路由，且并没有一个完善的路由器端针对 https 去广告的插件。
  2. 访问高安全性网站，如网银，可能因为 https 证书原因而被拒绝访问。

所以使用浏览器播放，加上广告屏蔽插件 uBlock Origin 即可。

至于 uBlock Origin 需要添加哪些订阅规则，我忘了，不过谷歌一下就有很多朋友整理出来的，只管添加就好。

效果

  • 无音频广告
  • 无视频广告
  • 无限切歌
  • 可顺序播放
  • 不能下载音乐（这个无法破解）

总结

在版权意识越来越强的当下，除了充会员及买单曲外，总是有另外的选择的。 Spotify 作为全球最大的流媒体平台，对于国内用户来说，体验不算太好，但也差强人意。很多国内小众音乐人的作品并没有被收录，相对的，国内平台也没有国外小众音乐人的作品。如果是像我这样没有特别爱好以及爱探索新音乐的用户，接受起来没有问题。最后，垄断总是危险的。愿世界和平。

在 Cloudflare 使用 Page Rules 重定向域名

前言

其实之前写的是用 Plesk 面板做重定向的，但考虑到

  • .tk 域名以后是不会再用的，保留其实是为了搜索引擎做跳转
  • Plesk 面板有流量限制（虽然我不可能被打）
  • 上了熊大免费的面板，如果被打的话，很不好意思

此外，直接添加 .tk 到 CF，其实是可以添加解析的，但不能使用动态 DNS 和自选 IP. 所以直接添加后，使用Page Rules 重定向就行了。Cloudflare 更新 Global API 已经一个月，如果他们想修复，是很容易的。但论坛中的工作人员没有做出回应（回复超过自己权限范围肯定不算回复啦），看这样子是拒绝白嫖党了。

通过 301 重定向，把 .tk 域名重定向到新域名

前言

继上一篇的 Cloudflare 拒绝 .tk 等一系列免费域名（好像真的有付费的 .tk, 但我没见过）之后，我决定扔掉 .tk 了。当然，像我这么穷的人，自然又去找了免费域名 .pp.ua, 来自乌克兰的免费域名，同时又去申请了 .eu.org 的域名。搞笑的是，.pp.ua 直接全域被加入到广告网站屏蔽列表里了，申请之后打不开，是排除到后来才发现的。在等待 .eu.org 域名获批前，暂时用着 .pp.ua 域名好了。

这篇文是通过更改 DNS NameServer 提供商，使用 URL Redirect (Permanent / 301) 来到达把旧域名重新定位到新的域名上，尽量减少搜索引擎蜘蛛访问错误而降权（其实也没几个访问）。

更新：直接添加 .tk 到 CF，是可以使用的，但不能使用动态 DNS 和自选 IP. 所以直接添加后，使用Page Rules 重定向就行了。
